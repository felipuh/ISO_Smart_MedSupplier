# Release Stage Positioning

Date: 2026-06-29

## Purpose

Clarify how to describe MedSupplier and the ecosystem at each commercial/release stage without overstating readiness.

| Stage | What It Means | Allowed Positioning | Not Allowed |
| --- | --- | --- | --- |
| Controlled Demo | Guided walkthrough using seeded or approved non-sensitive data. | Demo, workflow preview, buyer discovery. | Real production use claim. |
| Paid Pilot / Proof Of Concept | Limited customer scope with success criteria and controlled data. | Paid pilot, proof of concept, validation-ready review. | Broad enterprise rollout claim. |
| Controlled Enterprise MVP | Expanded workflow coverage with AdminApps authority, CI/CD, hardening, and validation package. | Controlled enterprise MVP, audit-ready package for review. | Regulated production approval claim. |
| Release Candidate | Candidate build/package after all RC gates pass and human review is pending or approved. | RC for controlled review. | Final production release unless approved. |
| Production-Like Non-Regulated | Approved environment for non-regulated operational use. | Production-like non-regulated deployment. | Regulated production validation claim. |
| Regulated Production | Requires customer/company validation, SOPs, training, environment controls, and release approval. | Only after formal approval by responsible organization. | Any claim before approval. |

## Required Phrase

Use: validation-ready and audit-ready; requires formal validation by the customer/company; does not replace SOPs or regulatory validation.
