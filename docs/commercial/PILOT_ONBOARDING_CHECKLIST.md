# Pilot Onboarding Checklist - ISO Smart MedSupplier

Date: 2026-06-29

## Commercial Setup

- [ ] Pilot owner named.
- [ ] Buyer owner named.
- [ ] Support owner named.
- [ ] Signed scope and limitations.
- [ ] Success criteria accepted.
- [ ] Pilot timeline accepted.
- [ ] Known limitations disclosed.

## AdminApps Setup

- [ ] Organization created or confirmed.
- [ ] `MEDSUPPLIER` product entitlement enabled.
- [ ] Subscription/billing state active or explicitly pilot-approved.
- [ ] Required users created.
- [ ] Roles assigned.
- [ ] Deny-safe behavior understood for non-entitled or billing-blocked states.
- [ ] No-fallback smoke scheduled before kickoff.

## MedSupplier Setup

- [ ] Tenant organization mapped to AdminApps external id.
- [ ] Supplier account scope configured.
- [ ] Customer account scope configured.
- [ ] Supplier roles configured.
- [ ] Customer roles configured.
- [ ] Demo or approved pilot data loaded.
- [ ] E2E smoke run on kickoff day.

## Data And Security

- [ ] Data classification approved.
- [ ] No real secrets included in pilot data.
- [ ] No patient or regulated production data unless separately approved.
- [ ] Private commercial data handling accepted.
- [ ] Access review completed.

## Operations

- [ ] Backup plan known.
- [ ] Restore expectations explained.
- [ ] Rollback owner named.
- [ ] Monitoring/logging expectations explained.
- [ ] Incident channel confirmed.

## Closeout

- [ ] Pilot findings reviewed.
- [ ] Success criteria evaluated.
- [ ] Open risks documented.
- [ ] Next decision recorded.
